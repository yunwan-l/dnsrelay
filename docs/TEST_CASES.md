# 建议测试用例

下面这些测试最好都做，并保存截图或日志，后面写报告和验收都用得上。

## 1. 本地命中测试

### 操作

```bat
nslookup www.baidu.com 127.0.0.1
```

前提是 `dnsrelay.txt` 中确实配置了 `www.baidu.com`。

### 预期结果

- 程序日志显示“本地命中”
- 返回的是配置文件里的 IP

## 2. 拦截测试

### 操作

```bat
nslookup test0 127.0.0.1
```

或者换成你们自己准备的、写在配置文件中的拦截域名。

### 预期结果

- 程序日志显示“本地拦截”
- 客户端收到域名不存在或查找失败
- 响应码为 `NXDOMAIN`

## 3. 中继测试

### 操作

```bat
nslookup www.bupt.edu.cn 127.0.0.1
```

要求该域名不在本地表中。

### 预期结果

- 程序日志显示“中继查询”
- 能收到上游 DNS 返回的结果

## 4. 并发测试

### 操作

快速发起多个查询，例如连续执行：

```bat
nslookup www.baidu.com 127.0.0.1
nslookup www.qq.com 127.0.0.1
nslookup www.bing.com 127.0.0.1
```

也可以在多个终端同时发起。

### 预期结果

- 程序不中断
- 多个查询都能被处理
- 不会因为第一个未返回就卡死后面的请求

## 5. 超时测试

### 操作

启动时指定一个不可用或不存在的上游 DNS：

```bat
dnsrelay -d 10.255.255.1
```

然后查询一个本地表中不存在的域名。

### 预期结果

- 程序日志显示“上游DNS超时”
- 客户端最终收到 `SERVFAIL`
- 超时条目会从映射表中清理掉

## 6. 迟到响应测试

### 思路

先人为制造超时，再观察后续迟到包是否会被程序丢弃。

### 预期结果

- 程序识别为“迟到响应”
- 不会误发给错误的客户端

## 7. 非 A 类型测试

### 操作

用系统或工具查询 AAAA 记录，例如：

```bat
nslookup -type=AAAA www.baidu.com 127.0.0.1
```

### 预期结果

- 如果本地命中但不是 A 类型，程序返回空回答而不是构造错误 A 记录
- 程序不崩溃

## 8. 缓存影响测试

### 操作

```bat
ipconfig /flushdns
ipconfig /displaydns
```

### 说明

很多时候“程序明明改了但查询结果没变”，其实是系统 DNS 缓存造成的。验收前一定记得清缓存。
