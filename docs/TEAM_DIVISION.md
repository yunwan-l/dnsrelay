# 三人分工建议

老师要求是 3 人一组，而且程序必须“所有同学都能消化”。  
所以分工应该是“主责不同”，不是“只会自己那一块”。

## 方案一：按代码模块分工

### 成员 A：主控与网络框架

负责：

- `dnsrelay.c`
- Socket 初始化
- 53 端口绑定
- `select()` 事件循环
- 启动参数处理

答辩时应重点能讲：

- 为什么使用单 socket
- 为什么使用 `select()`
- 为什么这不是忙等待

### 成员 B：DNS 协议解析

负责：

- `dns_protocol.c/.h`
- DNS 头部结构体
- QNAME 解析
- Question 段解析
- 本地响应构造

答辩时应重点能讲：

- DNS 报文 5 个部分
- `QNAME/QTYPE/QCLASS`
- `NXDOMAIN` 和 `SERVFAIL` 的区别

### 成员 C：中继与本地表

负责：

- `domain_table.c/.h`
- `relay_engine.c/.h`
- ID 转换
- 超时处理
- 迟到响应丢弃

答辩时应重点能讲：

- 为什么需要 TID 映射表
- 超时后为什么要删映射
- 为什么迟到响应不能乱发

## 方案二：按“开发 + 测试 + 文档”穿插分工

如果你们担心有人只会文档不会代码，可以这样做：

1. 成员 A 主写代码框架，同时负责最终集成。
2. 成员 B 主写协议解析，同时负责抓包分析。
3. 成员 C 主写测试和报告，但必须同时负责一块核心代码，例如本地表或超时模块。

## 最重要的一条

每天都要做一次交叉讲解：

1. A 给 B/C 讲主循环。
2. B 给 A/C 讲 DNS 报文。
3. C 给 A/B 讲 TID 映射和超时处理。

这样最后验收时，老师问谁都不至于卡死。
